## Session Notes

### Session 1

- Goal: Install JDK 21 and set up VS Code for Java development on Windows.
- Output completed: `java -version` and `javac -version` work with JDK 21.0.11 LTS, and the required Java extensions are installed in VS Code.
- What I understood: JDK is needed for both running and compiling Java programs, and VS Code needs Java extensions for a complete beginner-friendly workflow.
- What is still confusing: none for now
- What to review next: the difference between JDK, JRE, `java`, and `javac`

### Session 2

- Goal: Create the first Java program and understand the basic compile and run workflow.
- Output completed: `Hello.java` was created, compiled with `javac Hello.java`, and run successfully with `java Hello`, printing `Hello, Java!`.
- What I understood: `javac` compiles `.java` source code into a `.class` file, and `java` runs the compiled class.
- What is still confusing: none for now
- What to review next: why the class name, file name, and `java Hello` command must match

### Session 3

- Goal: Practice variables, basic data types, console output, and user input with `Scanner`.
- Output completed: `Basics1.java` runs successfully, reads `String`, `int`, and `double` input, and prints the values back to the console.
- What I understood: `String` stores text, `int` stores whole numbers, `double` stores decimal numbers, and `Scanner` reads user input from the console.
- What is still confusing: none for now
- What to review next: the difference between `nextLine()`, `nextInt()`, and `nextDouble()`, especially when mixing them in one program
