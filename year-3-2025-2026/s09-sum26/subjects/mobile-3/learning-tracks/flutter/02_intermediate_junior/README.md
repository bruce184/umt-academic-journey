# Phase 2: Intermediate / Junior

This phase moves from "the app works" to "the app is structured, testable, and more reliable". The focus is architecture, persistence, adaptive UI, testing, and release discipline.

## Modules

- `01_dart_code_quality/`
- `02_architecture_state_flow/`
- `03_persistence_caching/`
- `04_advanced_ui_navigation/`
- `05_testing_reliability/`
- `06_junior_capstone/`

## Expected Output

By the end of this phase, the goal is to have a junior-level Flutter app with a clearer architecture, persistence or API flow, better UI behavior across sizes, and baseline testing.
